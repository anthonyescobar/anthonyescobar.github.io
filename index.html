Three Partition Solver

Algorithm created and implemented by me. A function to check if a list of 3n size of numbers can be partitioned into a list of n triples such that the sum of each triple was the same. This program can handle a list of size 84 before throwing overflow errors due to the factorial nature of the problem. A simpler but brute force solution exists in Python using intertools, however I created a custom solution in Java without using an intertools equivalent.

ex.
given:
[2,4,8,12,15,2,0,6,3,2,9,1] as 2 4 8 12 15 2 0 6 3 2 9 1 .

ThreePartitionSolver will output true because you can partition this list as follows:
[(4,3,9),(15,0,1),(8,2,6),(2,2,12)].

while the output would be false for:
[6, -1, 8, 3455, 11, 7].

The output is also always false if the length of the input list is not a multiple of 3.

Make Optimal Change

When given any amount and denominations, find the most optimal way for making the given amount using the given denominations. Optimal i this context is using the least amount of coins.
ex.:
java MakeOptimalChange 4,1,9 12
12 cents can be made with 3 coins as follows:
- 3 4-cent coins
- 0 1-cent coins
- 0 9-cent coins

When no such way exists, IMPOSSIBLE is returned.
ex.:
java MakeOptimalChange 2,16,8 11
It is impossible to make 11 cents with those denominations.

Base Converter

This program converts a given number from one base to another.

  • args[0] represents the starting number;
  • args[1] represents the starting base;
  • args[2] represents the target base.

We require two or three arguments: the starting number will be specified using a special notation (below); the starting and target bases must both be positive and will be given in decimal. The target base, if absent, defaults to decimal. Note that the "digits" in the starting number must be consistent with the starting base. All arithmetic is done using Java type long, whose largest value is java.lang.Long.MAX_VALUE.

In order to represent numbers from all possible bases, we employ a special notation: a string like [d1][d2]...[dk] represents a k-digit number whose high-order digit has value d1 (decimal), whose next digit has value d2 (decimal), etc. Here are some examples:

  • The number 10110 would be represented [1][0][1][1][0].
  • The number 3204124 would be represented [3][2][0][4][1][2][4].
  • The base three number that means forty-seven would be represented [1][2][0][2].
  • The hexadecimal number for which we ordinarily write B3C would be represented [11][3][12].

So, to convert from 314 base five to base three, we would run java BaseConverter [3][1][4] 5 3.
To convert BBA326CF from hexadecimal to base twenty, we would run java BaseConverter [11][11][10][3][2][6][12][15] 16 20.
If we run java BaseConverter [6][14] 26 14, the output should be [12][2].

The above-described special notation should also be used for outputting the target number.

Madlibs

Arguments are processed in assembly and piped to the C file to compute. This program will take arguments and insert them into the given statement at its index number.
ex.:
char* words_to_use[] = { "swim", "brilliant", "git" };
madlib_by_numbers("The 1 2 likes to 0 in the 1 moonlight.", 3, words_to_use)
Would result
"The brilliant git likes to swim in the brilliant moonlight."

This program supports up to 10 arguments to madlib (digits 0-9) and if the argmument length does not reach a given number then the number is unchanged.
